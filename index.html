What is Meteor?

Open Source platform for building apps in JS

Built to power the next generation apps

One language everywhere

Isomorphic/Universal APIs mean that the same code, written in the same language, can run on both client and server.

Browser and Mobile

A great experience both on the web and as a phone or tablet app.

Live updates

Data updates live on the screen as it changes. Users can collaborate seamlessly.

Unified package system

One world of packages that work identically on browsers, servers and mobile devices. (Pull in npm and Cordova packages too!) One build tool that builds all parts of your app.

Radically less code

Accomplish in 10 lines what would otherwise take 1000, thanks to a reactive programming model that extends all the way from the database to the user's screen.

File/Folder Structure

  • /server - Code only run on server
  • /client - Code only run on client
  • /anythingElse - Code runs on both the server and client
  • /public - static assets like pics

Load Order

  • /lib - Loaded before anything else
  • main.* files - Loaded after everything else
  • Everything else loads in alphabetical order based on file name

Otherwise unopinionated

Distributed Data Protocol (DDP)

REST for websockets

Example


// In /lib
Posts = new Meteor.Collection('posts')

// On the server
Meteor.publish('postsByAuthor', function(author) {
  return Posts.find({ flagged: false, author: author });
});

// On the client
Meteor.subscribe('postsByAuthor', 'bob-smith');

// On the client
Template.posts.helpers({
    posts: Posts.find();
    // Client will only see Posts that have been flagged false
    //   and Posts with the author bob-smith
});

Tracker


var rv1, rvHandle;

rv1 = new ReactiveVar(1);

rvHandle = Tracker.autorun(function() {
  var val = rv1.get();
  return console.log("ReactiveVar1 is ", val);
});
// > "ReactiveVar1 is 1"

rv1.set(4);
// > "ReactiveVar1 is 4"
rv1.set(5);
// > "ReactiveVar1 is 5"
rvHandle.stop();
rv1.set(20);
// >

Fibers

Allows for synchronous looking code on the back-end

Fibers


var GithubAPI = Meteor.require('github');
var ghapi = new GithubAPI({version: "3.0.0"});

Meteor.methods({
  getProfile: function(username) {
    ghapi.user.getFrom({user: username}, function(err, profile) {
      // How to return?
      // return profile; // This won't work!
    });

    // We need to return the profile from here!
  }
});

Fibers


function getUserProfile(req, callback) {
  ghapi.user.getFrom(req, callback);
}
var wrappedGetProfile = Meteor._wrapAsync(getUserProfile);

Meteor.methods({
  getProfile: function(username) {
    return wrappedGetProfile({user: username});
  }
});

// OR if you understand how .bind() works.

var wrappedGetProfile = Meteor._wrapAsync(
        ghapi.user.getFrom.bind(ghapi.user));
